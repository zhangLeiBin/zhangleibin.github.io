小注：（每次发布需修改package.json的版本号，修改规则如下）
  • 语义版本号分为X.Y.Z三位，分别代表主版本号、次版本号和补丁版本号。
  • 如果只是修复bug，需要更新Z位。
  • 如果是新增了功能，但是向下兼容，需要更新Y位。
  • 如果有大变动，向下不兼容，需要更新X位。
